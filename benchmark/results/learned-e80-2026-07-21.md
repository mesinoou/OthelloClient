# Learned evaluator e80 benchmark

## Purpose

The CUDA-trained six-pattern evaluator is compared with the handcrafted
baseline and Edax 4.6. These are preliminary engineering measurements, not
tournament ratings.

## Model

- Runtime file: `data/evaluation-tables.bin`
- SHA-256: `6E118C928729E003E89742D3B57FE6ED35FA9233A38DBE8AF852041EBEE39457`
- Training backend: PyTorch CUDA, NVIDIA RTX A5000
- Quantization: int16, score scale 6400, zero clipped entries
- Java model: format 1, divisor 1, maximum absolute bound 75962

## Method

- Java 21 on Windows, 20 logical processors available
- One search thread for every measured player
- Opening book disabled for every player
- Ten, twenty, or fifty unique random eight-ply openings
- Fixed opening seed: `20260721`
- Every opening is played twice with learned evaluator colors exchanged
- Edax pondering and opening book disabled
- Win/loss is determined by final disc count

The fixed-depth baseline run gives both evaluators the same depth and isolates
evaluation quality. The timed runs include evaluation speed in the result.
Edax levels are Edax strength settings, not equal-time controls.

The learned evaluator reaches about 0.6 ply less than the handcrafted
evaluator in the 100 ms run, but wins 19 of 20 games. The evaluation gain is
therefore larger than the measured speed penalty at this budget.

Edax results place the current engine near level 7 and clearly below level 8
in this opening suite. Level 7 is almost exactly even by game score and has a
slightly negative average margin. Level 6 has a positive point estimate, but
its interval still includes 50%, so superiority is not established.
Intervals are Wilson score intervals with draws counted as half a point; they
are included as an approximate orientation for these paired samples.

### Tournament-budget search check

One paired opening was also played against Edax level 7 with the client's
tournament settings: 8 seconds per move, four threads, and a maximum depth of
64. The learned player scored one win and one draw with an average margin of
+15.0. This two-game result is not a strength estimate.

- Average completed depth: 12.24
- Average learned move time: 5290.778 ms
- Learned throughput: 1,468,078 nodes/s
- Search-budget stops: 33
- Exact endgame moves: 18
- Illegal moves and GTP errors: zero

## Limitations

- The main strength benchmark uses 100 ms per move, not the tournament's
  8-second internal budget.
- The fifty random eight-ply openings are reproducible but are not a balanced
  professional opening suite.
- Only one paired opening was measured at the tournament budget.
- Search remains deterministic at fixed depth, while time-limited results can
  vary slightly with machine load.

The next strength report should use a balanced professional opening suite and
at least 20 paired games at the tournament budget.
